# Yandex Stack Audit

Пользователь запустил **/yandex**. Аудит четырёх сервисов с итоговым вердиктом.

## Сразу прочитать

- Rules: `agent-workflow`, `grill-me-gate`
- Skills: `grill-me`, `seo-technical`, `seo-local`

## Шаг 0: Проверить `.env` (до grill-me)

Прочитать **только наличие** переменных (не выводить значения токенов в чат):

| Сервис | Переменные |
|--------|------------|
| Вебмастер | `YANDEX_WEBMASTER_OAUTH_TOKEN`, опц. `YANDEX_WEBMASTER_HOST_ID` |
| Метрика | `YANDEX_METRIKA_OAUTH_TOKEN`, `YANDEX_METRIKA_COUNTER_ID` |
| Директ | `YANDEX_DIRECT_OAUTH_TOKEN` |
| Бизнес | `YANDEX_BUSINESS_ORG_ID`, опц. `YANDEX_BUSINESS_CARD_URL` — см. README часть 4 |

Файлы: `.env`, `.env.local` в корне проекта. Инструкция: `README.md`, часть 4.

Записать в отчёт секцию **Доступ к API** — какие сервисы `api` / `manual` / `skip`.

## Если токена нет — спросить (по одному сервису)

Перед фазой этого сервиса, если токена нет и ответ ещё не зафиксирован:

```text
**[Сервис]:** в .env нет `YANDEX_..._TOKEN`.

Как поступаем?
1. **Скрин** — пришлёте скриншот кабинета
2. **Токен** — добавите в .env (подскажу переменную)
3. **Пропустить** — только чеклист / публичные проверки

**Рекомендация:** …
```

- **Скрин** → ждать изображение, анализировать, не выдумывать цифры
- **Токен** → назвать переменную, ждать «добавил», перечитать `.env` (значения не логировать)
- **Пропустить** → фаза в режиме `manual`, в отчёте пометить `➖ API пропущен`

Не спрашивать повторно, если ответ уже в Grill-me log / отчёте.

## Grill-me (после шага 0)

Первое сообщение grill-me — **вопрос 1** (домен/цель). Минимум 5 ответов.

Зафиксировать в `docs/yandex-audits/YYYY-MM-DD-domain.md` (+ `templates/YANDEX_AUDIT.md`):

- Домен, счётчик, карточка Бизнес, Директ да/нет, бизнес-цель
- Таблица **Доступ к API** по сервисам
- Grill-me log

«По рекомендации» → follow-up с подтверждением.

## Фазы

### 0. Подготовка
Отчёт + план → ждать «ок».

### 1. Сайт
robots, sitemap, HTTPS, счётчик в коде, schema. `seo-technical`.

### 2. Вебмастер
- **api** — данные через API по токену из `.env`
- **скрин** — по скриншоту пользователя
- **manual/skip** — чеклист + ссылка на webmaster.yandex.ru

### 3. Метрика
- **api** — цели, счётчик через API + сверка с кодом
- **скрин** — цели/настройки со скрина
- **manual** — счётчик в HTML, dataLayer, UTM

### 4. Бизнес
Публичная карточка + NAP. API редко — в основном скрин или чеклист.

### 5. Директ
- **api** — структура кампаний через API
- **скрин/ручной** — UTM, цели, минус-слова по чеклисту

### 6. Сквозная согласованность
Телефон, цели, UTM, гео — 🔴/🟡/🟢

### 7. Вердикт
Таблица /10 по сервисам + backlog (макс. 10).

## Безопасность

- В отчёте: `токен: ✅ задан` / `❌ нет`, без значений
- `.env` в `.gitignore`
